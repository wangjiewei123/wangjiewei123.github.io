2022.11.10问题总结

新知识,Hexo

一、认真看教程真的很重要

当我尝试hexo deploy命令时git报错
error：RPC failed;curl 56 OpenSSL SSL_read: Connectionwas reset,errno 10054
百度结果是：SSL证书问题或者git中版本问题
从网上查到的解决办法：
    删除掉.deploy_git重新clean、g、d
但是尝试多次后并没有效果，后来通过删掉远程仓库，重新按照教程搞了一边并且后来又删除掉
本地的ssh key并重新生成，添加到github中最终解决了，原因可能是我在搭建过程中漏掉了
某个不起眼的环节导致部署失败，因为后来没有再出现过这个错误，所以并没有深究原因，所以
仔细看教程真的很重要！！！！！

顺便再记录一下几个以后会常用到的快捷命令
    hexo clean 清除hexo缓存  头三个都可以通过缩写首字母开头来简化命令如：hexo c
    hexo generate 生成文章
    hexo deploy 部署
    hexo new post name 新建文章名
    hexo new page name 新建页面名

二、推荐使用Typora软件编辑博客！

hexo与Typora的完美结合..

Typora是我觉得非常好的Markdown文本编辑器可以拿来做笔记本身支持识别JAVA代码，但是需要收费，Typora对于插入图片的支持做得非常好，在文件->偏好设置进入设置。

三、此外从网上找的几种插入图片办法

在 此网站上有详细教程 上有详细教程！ 此外CSDN博客也支持MD文件上传

src 链接

如果我要插入的图片，是一个外部的 src 链接地址，比如该图片存放在某网站上，那就使用 Markdown 默认的插入图片的方式，方法和插入链接很像，只是前面多了一个感叹号，如下：

![alt](https://ws3.sinaimg.cn/large/005BYqpgly1g29eohl7qhj31c00u0dkz.jpg)

本地绝对路径

当 Hexo 博客中只用到少量图片时，可以将图片统一放在 source/images 文件夹中，通过 Markdown 语法访问它们。

![alt](/images/test.jpg)

本地相对路径

图片除了可以放在统一的 images 文件夹中，还可以放在文章自己的目录中，文章的目录可以通过配置 _config.yml：

post_asset_folder: true
